About This Demand Forecasting Tool

This tool converts repeated cycle-count snapshots into true consumption, cleans the time series, classifies items by movement, and produces simple, transparent short-term forecasts. No backend, no storage, zero data leakage.

What the tool does

  • Detects all date columns automatically—even if unsorted or mixed format.
  • Computes movement using max(previous − current, 0).
  • Identifies replenishment jumps and invalid values.
  • Builds a clean time-series per SKU with working-day normalization.
  • Creates rolling 30/60/90-day usage windows.
  • Classifies SKUs (Active / Low-Movement / Dead).
  • Generates simple 3-point moving-average forecasts.

Working-day normalization

Not all days are equal. Weekends and Ontario statutory holidays are removed before calculating per-day usage:

usage per working day = quantity moved ÷ number of working days

Planning windows

Management mode allows switching between 30, 60, and 90-day adjusted usage. All recommendations respond instantly to the selected planning window.

Vendor lead time

If your file includes a vendor lead-time column, it is auto-used. If not, you can enter lead times directly in the management panel.

Privacy and data handling

• Data never leaves your browser.
• Nothing is uploaded, saved, or logged.
• The tool runs entirely client-side.

How to use

  • Prepare a CSV with SKU, description, vendor, and multiple date columns.
  • Upload the file using the selector at the top.
  • Start in Analyst mode to verify data quality.
  • Switch to Management mode for rolling windows & recommendations.
  • Click any SKU to inspect detailed history and usage patterns.
